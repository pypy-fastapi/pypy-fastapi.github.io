    Protocols and Structural Subtyping

    Protocols are a powerful feature introduced in Python’s typing module that enable structural subtyping (also known as “duck typing” in static type checking). Instead of checking if a class explicitly inherits from a base class or interface, protocols check if a class has the required methods and properties.

    What Are Protocols?

    A protocol defines a set of methods and properties that a type must implement to be considered a subtype of that protocol. This allows for more flexible and expressive type checking without requiring explicit inheritance.

    Defining a Protocol

    You define a protocol by subclassing typing.Protocol and specifying the required methods and properties.

    from typing import Protocol

class SupportsClose(Protocol):
    def close(self) -> None:
        ...

    Any class that has a close method with the correct signature will be considered a subtype of SupportsClose, even if it does not explicitly inherit from it.

    Using Protocols in Functions

    You can use protocols as type hints to accept any object that matches the protocol’s structure.

    def close_all(things: list[SupportsClose]) -> None:
    for thing in things:
        thing.close()

    This function accepts a list of any objects that have a close method.

    Example: Structural Subtyping

    from typing import Protocol

class SupportsClose(Protocol):
    def close(self) -> None:
        ...

def close_all(things: list[SupportsClose]) -> None:
    for thing in things:
        thing.close()

class File:
    def close(self) -> None:
        print("File closed")

class Socket:
    def close(self) -> None:
        print("Socket closed")

class NotClosable:
    def disconnect(self) -> None:
        print("Disconnected")

files_and_sockets = [File(), Socket()]
close_all(files_and_sockets)  # Works fine

not_closables = [NotClosable()]
# close_all(not_closables)  # Type checker will raise an error

    Protocols with Properties and Methods

    Protocols can specify properties and methods with precise signatures.

    from typing import Protocol

class SupportsLen(Protocol):
    def __len__(self) -> int:
        ...

    Any object with a __len__ method returning an int is compatible with SupportsLen.

    Protocols with Optional Members

    Protocols can include optional members using @runtime_checkable and typing.runtime_checkable.

    from typing import Protocol, runtime_checkable

@runtime_checkable
class SupportsClose(Protocol):
    def close(self) -> None:
        ...

    This enables isinstance checks at runtime:

    file = File()
if isinstance(file, SupportsClose):
    file.close()

    Summary

    Feature Description
    Protocols Define structural interfaces without explicit inheritance
    Structural Subtyping Types are compatible if they have required members
    Protocol class Base class to define protocols
    @runtime_checkable Allows runtime isinstance checks against protocols
    Flexible typing Enables duck typing with static type checking

    Protocols provide a flexible and powerful way to specify interfaces in Python, enabling static type checkers to enforce duck typing principles and improving code robustness and readability.
