    Validation Errors

    Pydantic provides detailed and helpful validation errors to help you identify and fix issues in your data models. This document covers common validation errors, how they occur, and examples of their usage.

    Overview

    • Pydantic raises a ValidationError whenever it encounters invalid data during model validation.
    • Validation errors contain detailed information about what failed, where, and why.
    • You should raise ValueError, AssertionError, or PydanticCustomError in validators to indicate validation failures; do not raise ValidationError directly.

    • Use the ValidationError methods to inspect errors:

      • errors() - returns a list of error details.
      • error_count() - returns the number of errors.
      • json() - returns a JSON representation of the errors.
      • str(e) - human-readable error message.

    Common Validation Error Types

    1. arguments_type

    Raised when the input to a function expecting a tuple, list, or dict is of an invalid type.

    from typing import NamedTuple
from pydantic import BaseModel, ValidationError

class MyNamedTuple(NamedTuple):
    x: int

class MyModel(BaseModel):
    field: MyNamedTuple

try:
    MyModel.model_validate({'field': 'invalid'})
except ValidationError as exc:
    # Access the first error's 'type'
    print(exc.errors()[0]['type'])
# Output: 'arguments_type'

    2. assertion_error

    Raised when an assert statement fails inside a validator.

    from pydantic import BaseModel, ValidationError, field_validator

class Model(BaseModel):
    x: int

    @field_validator('x')
    @classmethod
    def force_x_positive(cls, v):
        assert v > 0
        return v

try:
    Model(x=-1)
except ValidationError as exc:
    print(exc.errors()[0]['type'])
# Output: 'assertion_error'

    3. literal_error

    Raised when a value is not one of the expected literals.

    from typing import Literal
from pydantic import BaseModel, ValidationError

class Model(BaseModel):
    x: Literal['a', 'b']

Model(x='a')  # OK

try:
    Model(x='c')
except ValidationError as exc:
    print(exc.errors()[0]['type'])
# Output: 'literal_error'

    4. mapping_type

    Raised when an error occurs during validation related to the Mapping protocol methods.

    from collections.abc import Mapping
from pydantic import BaseModel, ValidationError

class BadMapping(Mapping):
    def items(self):
        raise ValueError()

    def __iter__(self):
        raise ValueError()

    def __getitem__(self, key):
        raise ValueError()

    def __len__(self):
        return 1

class Model(BaseModel):
    x: dict[str, str]

try:
    Model(x=BadMapping())
except ValidationError as exc:
    print(exc.errors()[0]['type'])
# Output: 'mapping_type'

    5. missing

    Raised when required fields are missing from the input.

    6. model_type

    Raised when the input to a model is not a valid instance or dict.

    from pydantic import BaseModel, ValidationError

class Model(BaseModel):
    a: int
    b: int

m = Model.model_validate({'a': 1, 'b': 2})
print(m)

print(Model.model_validate(m))

try:
    Model.model_validate('not an object')
except ValidationError as exc:
    print(exc.errors()[0]['type'])
# Output: 'model_type'

    7. multiple_argument_values

    Raised when multiple values are provided for a single argument in a function decorated with validate_call.

    from pydantic import ValidationError, validate_call

@validate_call
def foo(a: int):
    return a

try:
    foo(1, a=2)
except ValidationError as exc:
    print(exc.errors()[0]['type'])
# Output: 'multiple_argument_values'

    8. multiple_of

    Raised when a value does not satisfy a multiple_of constraint.

    Example of Complex Validation Errors

    from pydantic import BaseModel, Field, ValidationError, field_validator

class Location(BaseModel):
    lat: float = 0.1
    lng: float = 10.1

class Model(BaseModel):
    is_required: float
    gt_int: int = Field(gt=42)
    list_of_ints: list[int]
    a_float: float
    recursive_model: Location

    @field_validator('a_float', mode='after')
    @classmethod
    def validate_float(cls, value: float) -> float:
        if value > 2.0:
            raise ValueError('Invalid float value')
        return value

data = {
    'list_of_ints': ['1', 2, 'bad'],
    'a_float': 3.0,
    'recursive_model': {'lat': 4.2, 'lng': 'New York'},
    'gt_int': 21,
}

try:
    Model(**data)
except ValidationError as e:
    print(e)

    5 validation errors for Model
is_required
  Field required [type=missing, input_value={'list_of_ints': ['1', 2, 'bad'], 'a_float': 3.0, 'recursive_model': {'lat': 4.2, 'lng': 'New York'}, 'gt_int': 21}, input_type=dict]
gt_int
  Input should be greater than 42 [type=greater_than, input_value=21, input_type=int]
list_of_ints.2
  Input should be a valid integer, unable to parse string as an integer [type=int_parsing, input_value='bad', input_type=str]
a_float
  Value error, Invalid float value [type=value_error, input_value=3.0, input_type=float]
recursive_model.lng
  Input should be a valid number, unable to parse string as a number [type=float_parsing, input_value='New York', input_type=str]

    Accessing Error Details Programmatically

    try:
    Model(**data)
except ValidationError as e:
    errors = e.errors()
    for err in errors:
        print(f"Error type: {err['type']}")
        print(f"Location: {err['loc']}")
        print(f"Message: {err['msg']}")
        print(f"Input: {err.get('input')}")
        print(f"Context: {err.get('ctx')}")
        print(f"Documentation: {err.get('url')}")
        print()

    Summary

    Error Type Description
    arguments_type Invalid argument type passed to function
    assertion_error Assertion failed in validator
    literal_error Value not in expected literals
    mapping_type Error during mapping protocol call
    missing Required field missing
    model_type Input not a valid model or dict
    multiple_argument_values Multiple values for one argument in function
    multiple_of Value not multiple of specified number
