    Custom Error Messages

    Pydantic allows you to customize error messages for validation failures, giving you control over how validation errors are reported. This document explains how to define custom error messages using validators and constrained types.

    1. Using Validators with Custom Errors

    You can raise ValueError or TypeError with custom messages inside validators to provide specific error feedback.

    from pydantic import BaseModel, ValidationError, field_validator

class User(BaseModel):
    username: str

    @field_validator('username')
    @classmethod
    def username_must_be_alphanumeric(cls, v):
        if not v.isalnum():
            raise ValueError('Username must be alphanumeric')
        return v

try:
    User(username='invalid username!')
except ValidationError as e:
    print(e)

    Output:

    1 validation error for User
username
Username must be alphanumeric (type=value_error)

    2. Using constr with Custom Error Messages

    When using constrained types like constr, you can specify custom error messages by subclassing and overriding __get_validators__.

    from pydantic import BaseModel, ValidationError

class CustomStr(str):
    @classmethod
    def __get_validators__(cls):
        yield cls.validate

    @classmethod
    def validate(cls, v):
        if not isinstance(v, str):
            raise TypeError('string required')
        if len(v) < 5:
            raise ValueError('string too short, minimum length is 5')
        return v

class User(BaseModel):
    username: CustomStr

try:
    User(username='abc')
except ValidationError as e:
    print(e)

    Output:

    1 validation error for User
username
string too short, minimum length is 5 (type=value_error)

    3. Using @validator with always=True and Custom Messages

    You can use the @validator decorator with always=True to enforce validation even if the field is missing, and raise custom errors.

    from pydantic import BaseModel, ValidationError, validator

class User(BaseModel):
    email: str

    @validator('email', always=True)
    def email_must_contain_at_symbol(cls, v):
        if '@' not in v:
            raise ValueError('Email must contain "@" symbol')
        return v

try:
    User(email='invalidemail.com')
except ValidationError as e:
    print(e)

    Output:

    1 validation error for User
email
Email must contain "@" symbol (type=value_error)

    4. Raising Custom Exceptions in Validators

    You can define custom exception classes and raise them inside validators to provide more structured error handling.

    from pydantic import BaseModel, ValidationError, field_validator

class UsernameError(ValueError):
    pass

class User(BaseModel):
    username: str

    @field_validator('username')
    @classmethod
    def check_username(cls, v):
        if not v.isalnum():
            raise UsernameError('Username must be alphanumeric')
        return v

try:
    User(username='invalid username!')
except ValidationError as e:
    print(e)

    Output:

    1 validation error for User
username
Username must be alphanumeric (type=value_error)

    5. Customizing Error Messages for Constrained Types

    You can customize error messages for constrained types like conint, constr, etc., by subclassing them.

    from pydantic import BaseModel, conint, ValidationError

class PositiveInt(conint(gt=0)):
    @classmethod
    def __get_validators__(cls):
        yield from super().__get_validators__()
        yield cls.custom_error

    @classmethod
    def custom_error(cls, v):
        if v <= 0:
            raise ValueError('Value must be a positive integer')
        return v

class User(BaseModel):
    age: PositiveInt

try:
    User(age=-5)
except ValidationError as e:
    print(e)

    Output:

    1 validation error for User
age
Value must be a positive integer (type=value_error)
