    Settings Management

    Pydantic provides a convenient way to manage application settings and environment variables using the BaseSettings class. This allows you to define configuration models that automatically read from environment variables, .env files, and other sources.

    Defining Settings with BaseSettings

    To create a settings model, inherit from pydantic.BaseSettings instead of BaseModel:

    from pydantic import BaseSettings

class Settings(BaseSettings):
    app_name: str
    admin_email: str
    items_per_user: int = 50

settings = Settings()
print(settings.app_name)
print(settings.admin_email)
print(settings.items_per_user)
    • Fields are defined like in regular Pydantic models.
    • Values are loaded from environment variables by default.
    • You can provide default values as usual.

    Environment Variable Support

    BaseSettings automatically reads environment variables matching the field names in uppercase:

    export APP_NAME="My Awesome App"
export ADMIN_EMAIL="admin@example.com"
export ITEMS_PER_USER=100

    Running the above Python code after setting these environment variables will populate the Settings instance accordingly.

    Using .env Files

    You can load settings from a .env file by installing the python-dotenv package and placing a .env file in your project:

    APP_NAME=My Env App
ADMIN_EMAIL=envadmin@example.com
ITEMS_PER_USER=25

    BaseSettings will automatically read from the .env file if present.

    Customizing Environment Variable Names

    You can customize the environment variable names using Field with the env parameter:

    from pydantic import BaseSettings, Field

class Settings(BaseSettings):
    app_name: str = Field(..., env='MY_APP_NAME')
    admin_email: str = Field(..., env='MY_ADMIN_EMAIL')

    Now, MY_APP_NAME and MY_ADMIN_EMAIL environment variables will be used instead.

    Nested Settings

    You can nest settings models for better organization:

    from pydantic import BaseSettings

class DatabaseSettings(BaseSettings):
    host: str
    port: int

class Settings(BaseSettings):
    app_name: str
    database: DatabaseSettings

settings = Settings()
print(settings.database.host)

    Environment variables for nested models use __ (double underscore) as a separator by default:

    export DATABASE__HOST=localhost
export DATABASE__PORT=5432

    Secrets Management

    You can use Pydantic’s SecretStr and SecretBytes types to handle sensitive data securely:

    from pydantic import BaseSettings, SecretStr

class Settings(BaseSettings):
    db_password: SecretStr

settings = Settings()
print(settings.db_password)  # Will not print the secret value
print(settings.db_password.get_secret_value())  # To access the actual secret

    Example: Complete Settings Model

    from pydantic import BaseSettings, SecretStr, Field

class Settings(BaseSettings):
    app_name: str = Field(..., env='APP_NAME')
    admin_email: str = Field(..., env='ADMIN_EMAIL')
    items_per_user: int = 50
    db_password: SecretStr

settings = Settings()
print(settings.app_name)
print(settings.admin_email)
print(settings.items_per_user)
print(settings.db_password.get_secret_value())

    Summary

    Feature Description
    BaseSettings Base class for settings with env var support
    Environment Variables Automatically read from env vars
    .env File Support Load settings from .env files
    Custom Env Variable Names Use Field(..., env='VAR_NAME')
    Nested Settings Support for nested configuration models
    Secrets Management Use SecretStr and SecretBytes for secrets
