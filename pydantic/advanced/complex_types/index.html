    Complex Types

    Pydantic supports a wide range of complex data types out of the box, allowing you to model sophisticated data structures with validation and parsing. This guide covers some of the most commonly used complex types.

    1. Decimal

    Use decimal.Decimal for precise decimal arithmetic (e.g., financial calculations).

    from decimal import Decimal
from pydantic import BaseModel

class Product(BaseModel):
    price: Decimal

product = Product(price='19.99')
print(product.price)  # Output: 19.99
print(type(product.price))  # Output: <class 'decimal.Decimal'>

    2. UUID

    Use uuid.UUID for universally unique identifiers.

    from uuid import UUID
from pydantic import BaseModel

class Item(BaseModel):
    id: UUID

item = Item(id='123e4567-e89b-12d3-a456-426614174000')
print(item.id)  # Output: 123e4567-e89b-12d3-a456-426614174000
print(type(item.id))  # Output: <class 'uuid.UUID'>

    3. EmailStr and AnyUrl

    Pydantic provides specialized types for emails and URLs with validation.

    from pydantic import BaseModel, EmailStr, AnyUrl

class User(BaseModel):
    email: EmailStr
    website: AnyUrl

user = User(email='user@example.com', website='https://example.com')
print(user.email)    # Output: user@example.com
print(user.website)  # Output: https://example.com

    4. Path and FilePath

    Use Path and FilePath from pydantic to validate filesystem paths.

    from pydantic import BaseModel, FilePath
from pathlib import Path

class Config(BaseModel):
    log_file: FilePath
    data_dir: Path

config = Config(log_file='/var/log/app.log', data_dir='/usr/data')
print(config.log_file)
print(config.data_dir)

    5. Constrained Types

    Pydantic supports constrained types like conint, constr, conlist to enforce restrictions.

    from pydantic import BaseModel, conint, constr

class User(BaseModel):
    age: conint(gt=0, lt=120)  # age must be between 1 and 119
    username: constr(min_length=3, max_length=50)

user = User(age=30, username='alice')
print(user)

    6. Literal

    Use Literal to restrict a field to specific literal values.

    from typing import Literal
from pydantic import BaseModel

class Pet(BaseModel):
    type: Literal['cat', 'dog', 'bird']

pet = Pet(type='dog')
print(pet)

    7. Union and Optional

    Use Union and Optional for fields that can accept multiple types or be nullable.

    from typing import Union, Optional
from pydantic import BaseModel

class Item(BaseModel):
    id: Union[int, str]
    description: Optional[str]

item1 = Item(id=123)
item2 = Item(id='abc', description='A sample item')
print(item1)
print(item2)

    8. Tuple

    Use Tuple for fixed-length sequences with specified types.

    from typing import Tuple
from pydantic import BaseModel

class Point(BaseModel):
    coordinates: Tuple[float, float]

point = Point(coordinates=(10.5, 20.3))
print(point)

    Summary

    Type Description
    Decimal Precise decimal numbers
    UUID Universally unique identifiers
    EmailStr Validated email strings
    AnyUrl Validated URLs
    Path, FilePath Filesystem path validation
    conint, constr Constrained integer and string types
    Literal Restrict to specific literal values
    Union, Optional Multiple possible types or nullable fields
    Tuple Fixed-length typed sequences
